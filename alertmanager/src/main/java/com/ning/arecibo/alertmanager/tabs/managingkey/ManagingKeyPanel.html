Managing Rules Explained  |  What's the difference between action types: DISABLE and QUIESCE?
Explanation

A Managing Rule represents rules for disabling or quiescing alerts. A Managing Rule can be associated with one or more Alerting Configurations. Each associated Alerting Configuration will abide by the managing rule's settings.

If an Alerting Configuration has multiple Rules specified, alert suppression is invoked if any 1 of its Managing Rules is in effect.

A Managing Rule can have both scheduled or manual invocation options, in which case manual invocation always takes precedence.

To associate a Managing Rule with an Alerting Configuration, visit the Alerting Configurations tab.

Explanation

A Managing Rule with an action type of DISABLE, when invoked, will result in the managed alerting configurations to be completely disabled. No notifications will be sent, and no logging or state will be maintained.

A Managing Rule with an action type of QUIESCE, when invoked, will still allow the internal management and tracking of alerting status, with the exception that no notification will be sent. Thus, internal alert state will still be maintained, and currently active alerts will still be visible in the dashboard, etc.

If multiple Managing Rule's are in effect for a given alerting configuration, DISABLE actions take precedence over QUIESCE actions.
